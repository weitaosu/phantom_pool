# x402 SDK Reference (Solana x402 + TRON AI Bounties)

> Source: https://docs.cdp.coinbase.com/x402/welcome
> GitHub: https://github.com/coinbase/x402
> NPM: `@x402/express`, `@x402/core`, `@x402/evm`, `@x402/svm`, `@x402/fetch`
> Facilitator (testnet, no signup): https://x402.org/facilitator
> Facilitator (production, CDP): https://api.cdp.coinbase.com/platform/v2/x402

## What It Is

x402 is an open payment protocol using HTTP 402 "Payment Required" status codes. When an API request arrives without payment, the server responds with 402 + payment instructions. The client pays on-chain (USDC), retries with a payment header, and gets access. No API keys, no subscriptions.

## Protocol Flow

1. Client → `GET /api/data` → Server
2. Server → `402 Payment Required` + `PAYMENT-REQUIRED` header (base64 PaymentRequirements)
3. Client selects a PaymentRequirement, signs a payment payload
4. Client → `GET /api/data` + `X-PAYMENT` header → Server
5. Server verifies payment via facilitator → `200 OK`

## Packages

```bash
# Full server setup (Express)

# Full client setup (fetch-based)

# Other server frameworks

```

**IMPORTANT API VERSION NOTE**: There are TWO versions of the x402 Express middleware floating around in docs and tutorials. The NEWER version uses `x402ResourceServer` + scheme registration. The OLDER version uses a simpler 3-argument `paymentMiddleware(address, routes, facilitator)`. Check which version you're installing.

## Server Setup (Express) — NEW API (recommended)

```typescript
import express from "express";
import { paymentMiddleware, x402ResourceServer } from "@x402/express";
import { ExactEvmScheme } from "@x402/evm/exact/server";
import { HTTPFacilitatorClient } from "@x402/core/server";

const app = express();
const payTo = "0xYourReceivingAddress";

// Create facilitator client (testnet — no signup needed)
const facilitatorClient = new HTTPFacilitatorClient({
  url: "https://x402.org/facilitator"
});

// Create resource server and register payment schemes
const server = new x402ResourceServer(facilitatorClient)
  .register("eip155:84532", new ExactEvmScheme());  // Base Sepolia

app.use(
  paymentMiddleware(
    {
      "GET /api/data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.10",           // USD amount (maps to USDC)
            network: "eip155:84532",  // CAIP-2 network identifier
            payTo: payTo,
          },
        ],
        description: "Access to dark pool matching engine",
        mimeType: "application/json",
      },
    },
    server,
  ),
);

app.get("/api/data", (req, res) => {
  // Payment verified — serve content
  res.json({ message: "Paid content" });
});

app.listen(3001);
```

## Server Setup (Express) — OLD/SIMPLE API

Many tutorials and the npm package `x402-express` (without `@` scope) use this simpler API:

```typescript
import express from "express";
import { paymentMiddleware } from "x402-express";

const app = express();

app.use(paymentMiddleware(
  "0xYourReceivingAddress",
  {
    "GET /api/orders/submit": {
      price: "$0.10",
      network: "base-sepolia",         // or "solana-devnet"
      description: "Dark pool order submission",
    },
  },
  { url: "https://x402.org/facilitator" }
));

app.get("/api/orders/submit", (req, res) => {
  const paymentDetails = (req as any).x402Payment;
  res.json({ orderId: "...", txHash: paymentDetails?.transactionHash });
});

app.listen(3001);
```

## Client Setup (Node.js / Agent)

```typescript
import { wrapFetch } from "@x402/fetch";
import { ExactEvmScheme } from "@x402/evm/exact/client";

// Wrap fetch to auto-handle 402 responses
const payingFetch = wrapFetch(globalThis.fetch, {
  privateKey: process.env.AGENT_PRIVATE_KEY,
  schemes: [new ExactEvmScheme()],
});

// Use like normal fetch — payment handled automatically
const response = await payingFetch("https://api.darkpool.trade/api/orders/submit", {
  method: "POST",
  body: JSON.stringify({ market: "...", isYes: true, size: 1000 }),
});
```

## Solana (SVM) Support

x402 supports Solana via `@x402/svm`:

```typescript
import { ExactSvmScheme } from "@x402/svm/exact/server";

const server = new x402ResourceServer(facilitatorClient)
  .register("solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1", new ExactSvmScheme());
  // solana-devnet CAIP-2 identifier

// Route config
app.use(paymentMiddleware(
  {
    "GET /api/solana/orders/submit": {
      accepts: [{
        scheme: "exact",
        price: "$0.10",
        network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",  // solana-devnet
        payTo: "YourSolanaAddress...",
      }],
      description: "Solana dark pool access",

));
```

## Custom x402-style for TRON (Not Natively Supported)

x402 does NOT natively support TRON. The plan calls for a custom middleware that mimics the 402 flow. Implementation:

```typescript
import { TronWeb } from 'tronweb';

const tronWeb = new TronWeb({ fullHost: 'https://nile.trongrid.io' });

function tronX402Middleware(requiredAmount: string, receivingAddress: string) {
  return async (req, res, next) => {
    const txHash = req.headers['x-tron-payment-txhash'];

    if (!txHash) {
      return res.status(402).json({
        payment_required: {
          chain: 'TRON-NILE',
          amount: requiredAmount,
          asset: 'USDT_TRC20',
          to: receivingAddress,
          memo: 'dark-pool-access',
        }
      });
    }

    // Verify TRC-20 USDT transfer on-chain
    try {
      const txInfo = await tronWeb.trx.getTransactionInfo(txHash);
      // Verify: correct amount, correct recipient, correct token
      // TRC-20 transfers are in the `log` field of transaction info
      req.payment = { txHash, verified: true };
      next();
    } catch (err) {
      return res.status(402).json({ error: 'Payment verification failed' });
    }
  };
}
```

## Gotchas & Risks

1. **Two API versions**: `@x402/express` (scoped, new API with schemes) vs `x402-express` (unscoped, simple API). They have different function signatures. Pick one and stick with it.
2. **Facilitator required**: Payment verification goes through a facilitator server. Use `https://x402.org/facilitator` for testnet (no signup). For production, use CDP facilitator with API keys.
3. **TRON not supported**: Must build custom middleware. The 402 response format is your own design.
4. **Testnet USDC needed**: Clients need testnet USDC on Base Sepolia or Solana Devnet to make payments.
5. **Latency**: Round-trip for a payment (request → 402 → pay → retry → 200) takes 2-4 seconds on Base.
6. **Free tier**: CDP facilitator gives 1,000 free transactions/month.
7. **The plan references `@x402/express` and `@x402/svm`** — verify these are the correct current package names before installing.
